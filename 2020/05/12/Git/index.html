Git的基本操作

1.创建版本库

  • git init

2.新建文件

  • vim 文件名.文件类型：新建文件并进入文件（例：git first.txt）
  • alt+i：进入文件编辑模式
  • esc+:wq：退出编辑模式并保存文件

3.提交文件

  • git status：查看文件状态
  • git add 文件名：将文件添加到暂存区
  • git commit -m "注释内容"：提交文件到本地库（带注释）

4.查看文件提交记录

  • git log 文件名：查看历史记录
  • git log --pretty=oneline 文件名：简易信息查看

5.回退历史

  • git reset --hard HEAD^：回退到上次提交
  • git reset --hard HEAD~n：回退n次操作

6.版本穿越

  • git reflog 文件名：查看历史记录的版本号
  • git reset --hard 版本号：执行操作

7.还原文件

  • git checkout -- 文件名：还原文件

8.删除某个文件

  • rm -rf 文件名：删除文件（此时未真正删除）
  • git add 文件名：提交
  • git commit -m "注释内容"

Git分支操作

1.创建分支

  • git branch 分支名：创建分支
  • git branch -v：查看分支

2.切换分支

  • git checkout 分支名：切换分支
  • git checkout -b 分支名：一步完成创建并切换分支

3.合并分支

  • git checkout master：先切换到主分支
  • git merge 副分支名：合并分支

4.删除分支

  • git checkout master：先切换到主分支
  • git branch -D 分支名：删除副分支

5.解决冲突

  • git diff：找到发生冲突的文件及冲突的内容
  • git add 文件名+git commit -m "注释内容"：修改冲突文件内容后提交（后缀MERGING消失，说明冲突解决）

远程Git操作

1.搭建代码库

(1) git init
(2) git config

2.提交代码

(1) git add 文件名
(2) git commit -m "注释内容"

3.GitHub准备工作

(1) 准备GitHub账号
(2) 在GitHub搭建项目

4.推送代码到远端

(1) git remote add origin <url>
(2) git push origin master

5.远端操作

(1) git clone <url>：克隆到远端本地库
(2) git add 文件名
(3) git commit -m "注释内容"
(4) git push origin master

6.本地拉取远端提出的修改请求

(1) git pull：拉取远端代码
(2) 查看并判断是否通过该请求

7.协作冲突

(1) git pull：拉取远端文件
(2) 修改合并
(3) git add
(4) git commit -m "注释内容"

GitFlow工作流

  • master ：主干分支
  • develop ：开发分支
  • hotfix ：bug修理分支
  • release ：发布版本分支
  • feature ：功能分支
